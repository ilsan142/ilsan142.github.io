oh-my-claudecode

Claude Code용 멀티 에이전트 오케스트레이션 프레임워크 — 한국어 사용 가이드

1. 소개

oh-my-claudecode (OMC) 는 Anthropic의 Claude Code CLI 위에서 동작하는 멀티 에이전트 오케스트레이션 도구입니다. 작업을 받으면 플래너 → PRD → 실행자 → 검증자 → 수정자 라는 단계형 파이프라인으로 자동으로 흘려보내고, 필요하면 Codex / Gemini 같은 다른 모델까지 함께 호출해 병렬로 일을 시킵니다.

Team Orchestration

단계형 파이프라인으로 계획→실행→검증→수정 자동화

Smart Model Routing

간단한 작업은 Haiku, 복잡한 추론은 Opus로 자동 라우팅

CLI Workers

tmux에 Codex / Gemini 프로세스를 띄워 병렬 작업

Custom Skills

반복되는 디버깅 패턴을 스킬로 추출, 자동 주입

Rate Limit 자동 재개

한도 풀리면 세션 자동 이어붙이기

Cost Tracking

토큰 사용량을 세션 단위로 기록·분석

공식 저장소: github.com/Yeachan-Heo/oh-my-claudecode

2. 사전 요구사항

  • Claude Code CLI 필수
  • Claude Max/Pro 구독 또는 Anthropic API 키 필수
  • tmux — team 모드, rate-limit 대기, CLI 워커에 필요 권장
  • Gemini CLI / Codex CLI — 멀티 모델 워크플로우용 선택

OS별 tmux 설치 — install tmux per OS

# macOS
brew install tmux

# Ubuntu / Debian / WSL
sudo apt install tmux

# Windows (네이티브, psmux)
# https://github.com/marlocarlo/psmux

선택 — Gemini / Codex CLI 설치

npm install -g @google/gemini-cli
npm install -g @openai/codex

3. 설치 (Plugin Marketplace · 권장)

중요

아래 두 명령은 Claude Code 세션 내부에서 직접 입력해야 합니다. 터미널 셸이 아니라, claude 를 실행한 뒤 프롬프트에 그대로 붙여넣으세요. 

/plugin marketplace add https://github.com/Yeachan-Heo/oh-my-claudecode
/plugin install oh-my-claudecode

대안: npm 전역 설치

플러그인 마켓플레이스 대신, 셸에서 바로 설치할 수도 있습니다 — 패키지명이 oh-my-claude-sisyphus 인 점에 주의.

npm i -g oh-my-claude-sisyphus@latest

설치 후 omc 명령이 PATH에 추가됩니다.

4. 초기 설정

설치 직후 한 번 실행해 기본 컴포넌트(에이전트·스킬·MCP)와 설정을 동기화합니다.

# Claude Code 안에서
/omc-setup
/oh-my-claudecode:omc-setup

# 또는 터미널에서
omc setup
omc setup --force          # 이미 설치된 경우 강제 재설치
omc config --validate      # 설정 유효성 검사
omc info                   # 설치된 에이전트 / 스킬 / MCP 목록 보기

현재 환경 점검

omc doctor                 # 진단 (설정·의존성 체크)
omc doctor conflicts       # 충돌만 따로 확인
omc version                # 상세 버전 정보
omc update --check         # 업데이트 확인

5. 매직 키워드 (실행 모드)

OMC의 핵심 사용법은 매직 키워드입니다. Claude Code 프롬프트에 자연어처럼 키워드를 섞어 쓰면, OMC가 해당 모드의 파이프라인을 자동으로 가동합니다.

키워드역할예시
autopilot아이디어 → 완성 코드까지 완전 자율 실행autopilot build a React dashboard
ralph검증 통과할 때까지 끈질기게 반복 — Persistence 모드ralph refactor the API
ulw (ultrawork)최대 5개 이상 에이전트 병렬 — 대량 수정용ulw fix all typescript errors
team N:roleN명 에이전트로 팀 파이프라인 가동team 5:executor refactor backend
plan대화형 계획 인터뷰plan the auth system
ralplan합의(consensus) 기반 반복 계획 수립ralplan this feature
/ccgClaude+Codex+Gemini 3모델 동시 자문/ccg review this module
ask codex / ask gemini다른 모델에게 한 번만 자문ask codex to review this
deslopAI 흔적 / 군더더기 제거deslop this file
/release릴리스 매니지먼트 워크플로우/release

7. omc CLI (터미널)

tmux 기반의 셸 진입점입니다. Claude Code 외부에서 OMC를 제어할 때 사용합니다.

기본 실행

omc                        # tmux 세션에서 Claude Code 실행
omc launch                 # 위와 동일

세션 / 워크트리

omc teleport add my-branch        # 워크트리 생성
omc teleport list                 # 워크트리 목록
omc teleport remove ./path        # 워크트리 제거
omc teleport 1234                 # 이슈/PR 번호 또는 URL로 자동 워크트리

Rate Limit 자동 대기

omc wait                 # 현재 한도 상태 확인
omc wait --start         # 한도 풀리면 자동 재개하는 데몬 시작
omc wait daemon start    # 데몬 직접 제어
omc wait daemon stop
omc wait detect          # 막힌 세션 스캔
omc wait status          # 상세 상태

HUD / 상태선

omc hud                       # 현재 상태선 출력
omc hud --watch               # 실시간 갱신
omc hud --watch --interval 500

상호운용 (Codex와 분할 화면)

omc interop                   # tmux 분할 — Claude + Codex 동시 사용

8. 실행 플래그

플래그효과
--notify false이번 세션 알림 전부 끄기
--madmax / --yolo모든 권한 프롬프트 스킵 위험
--telegramTelegram 알림 켜기
--discordDiscord 알림 켜기
--slackSlack 알림 켜기
--webhook웹훅 알림 켜기

주의

--madmax / --yolo 는 파괴적인 작업까지 자동 승인합니다. 신뢰할 수 있는 격리 환경(컨테이너·워크트리)에서만 쓰세요.

9. 추천 워크플로우

① 명확한 작업 — No-Brainer

/autopilot  →  /ultrawork  →  /ralph
# 빠른 자율 실행. 작업 의도가 분명할 때.

② PRD부터 시작 — Full-Auto

/ralplan  →  /teams (또는 /omc-teams)  →  /ralph
# 합의 계획 → 병렬 에이전트 → 완료 검증

③ 버그 잡기 — Fix/Debug

/plan  →  /ralph  →  /ultraqa
# 원인 분석 → 반복 수정 → 엔드투엔드 테스트

④ 여러 티켓 동시 처리 — Parallel Issues

/omc-teams N:architect  →  parallel workers  →  /ralplan  →  /ralph + /ultrawork  →  /ultraqa
# 워크트리별로 다중 티켓 동시 처리

10. 알림 & HUD

장시간 실행되는 자율 모드(autopilot/ralph)는 푸시 알림과 같이 쓸 때 진가가 나옵니다.

# Telegram 채널 설정 예시
omc config-stop-callback telegram

# Discord
omc config-stop-callback discord

# Slack
omc config-stop-callback slack

# 알림 프로필 관리
omc config-notify-profile --list
omc config-notify-profile my-profile --show

HUD(상태선)는 작업 진행률·토큰·모델·에이전트 수를 실시간으로 보여줍니다.

omc hud --watch --interval 1000

11. 실전 팁

  • 모호하면 /deep-interview 부터. 요구사항을 자동으로 파헤쳐 PRD로 굳혀 줍니다.
  • 비용은 라우팅으로 절감. 단순 작업은 자동으로 Haiku로 내려가니, 무조건 autopilot 만 외치는 것보다 적절한 키워드 선택이 30~50% 토큰을 아낍니다.
  • omc wait --start 는 켜두자. Max/Pro 한도에 걸려도 풀리는 즉시 자동으로 이어줍니다.
  • 큰 리팩토링은 워크트리에서. omc teleport add 로 분리한 뒤 ulw 로 병렬 처리.
  • 반복 디버깅은 /learner 로 스킬화. 같은 실수를 다음 세션에서 안 반복하게 만들어 줍니다.
  • Codex·Gemini는 리뷰 용도로 가성비 좋음. 코딩은 Claude, 검토는 /ccg 또는 ask codex.

12. 트러블슈팅

문제

/plugin install 후에도 명령이 안 보임

→ Claude Code를 한 번 종료하고 재실행. 그래도 안 되면:

/omc-doctor
# 또는
omc doctor
문제

team / ralph 모드에서 tmux 관련 오류

→ tmux가 설치돼 있고 PATH에 잡혀 있어야 합니다.

which tmux && tmux -V
# Linux/WSL 미설치 시:
sudo apt install tmux
문제

npm 설치 시 권한 오류 (EACCES)

# nvm 사용 권장. sudo 설치는 가능하면 피하기.
# 어쩔 수 없으면:
sudo npm i -g oh-my-claude-sisyphus@latest
한도 도달

갑자기 작업이 멈추면 Max/Pro 한도 가능성

omc wait status
omc wait --start    # 자동 재개 데몬

문서가 빠르게 업데이트되므로 최신 명령은 항상 공식 docs 및 저장소 README 를 함께 확인하세요.
